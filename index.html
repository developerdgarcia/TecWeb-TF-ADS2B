3) Quais outros protocolos de comunicação para troca de mensagens via internet existem? Pesquise e cite 3 exemplos, descrevendo sua função e aplicação.

  • PROTOCOLO 1: TELNET - Trata-se de um protocolo de uso geral cujo objetivo e permitir a conexao, via terminal, de um cliente remoto em um dado servidor. Ele permite que usuarios remotos possam estabelecer uma conexao TCP com um servidor. Por padrao, os servidores utilizam a porta 23 para conexoes TELNET.

  • PROTOCOLO 2: FTP - O aplicativo ftp foi um dos primeiros aplicativos desenvolvidos para uso no que hoje chamamos de internet. Surgiu para permitir a troca de arquivos entre computadores ligados em rede. O protocolo FTP suporta este aplicativo. O protocolo FTP permite a troca de arquivos nos formatos ASCII e binario. O protocolo FTP permite a troca de arquivos nos formatos ASCII e binario.

  • PROTOCOLO 3: SMTP - O protocolo SMTP (Simple Mail Transfer Protocolo ou protocolo simples de transferência de correio) é um dos protocolos que suportam o processo de envio de mensagens eletronicas via internet. Definido pela RFC 821 este protocolo utiliza o TELNET para autenticacao e, uma vez que o cliente seja autenticado, realiza uma sequencia de comandos para o envio do correio eletronico (e-mail) para o servidor que, por sua vez, efetuara a entrega ao destinatario ou fara o roteamento para o servidor deste usuario.

5) Pequisem sobre as limitacoes do HTTP 1.1 e como o HTTP2 vence essa limitacao, cite 2 dessas limitacoes e suas respostas no HTTP2

O HTTP 1.1 e limitado em questao de seguranca e limitacao de Request, ja o HTTP2 em relacao a seguranca e SSL em default, todo servidor que for trabalhar com HTTP2, obrigatoriamente vai ser HTTPS em qualquer estado de comunicacao. Já em relacao ao Request o HTTP2 transcedem as limitacoes atuais para multiplos requests simultaneos, sem esperar a resposta do servidor de request bloqueantes, nao sendo mais necessario fazer concatenacao de arquivos
